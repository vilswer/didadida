# 视频服务接口测试文档

## 1. 用户服务接口

### 1.1 发送注册验证码
- **请求方式**：POST
- **URL**：`http://localhost:8001/user/send-captcha`
- **参数**：
  - Query参数：`phone`（手机号）
- **预期响应**：
  ```json
  {
    "code": 200,
    "message": "操作成功",
    "data": null
  }
  ```
- **测试步骤**：
  1. 输入有效的手机号
  2. 发送请求
  3. 验证返回状态码为200
  4. 检查手机是否收到验证码

### 1.2 用户注册
- **请求方式**：POST
- **URL**：`http://localhost:8001/user/register`
- **参数**：
  - Body参数：
    ```json
    {
      "phone": "13800138000",
      "password": "123456",
      "captcha": "123456",
      "nickname": "测试用户"
    }
    ```
- **预期响应**：
  ```json
  {
    "code": 200,
    "message": "操作成功",
    "data": 1001
  }
  ```
- **测试步骤**：
  1. 先发送验证码
  2. 填写注册信息，包括手机号、密码、验证码、昵称
  3. 发送请求
  4. 验证返回状态码为200，返回用户ID

### 1.3 用户登录
- **请求方式**：POST
- **URL**：`http://localhost:8001/user/login`
- **参数**：
  - Body参数：
    ```json
    {
      "phone": "13800138000",
      "password": "123456"
    }
    ```
- **预期响应**：
  ```json
  {
    "code": 200,
    "message": "操作成功",
    "data": {
      "userId": 1001,
      "nickname": "测试用户",
      "token": "eyJhbGciOiJIUzI1NiJ9..."
    }
  }
  ```
- **测试步骤**：
  1. 输入手机号和密码
  2. 发送请求
  3. 验证返回状态码为200，返回用户信息和token

### 1.4 用户退出登录
- **请求方式**：POST
- **URL**：`http://localhost:8001/user/logout`
- **参数**：
  - Header参数：`Authorization`（登录时获取的token）
- **预期响应**：
  ```json
  {
    "code": 200,
    "message": "操作成功",
    "data": null
  }
  ```
- **测试步骤**：
  1. 先登录获取token
  2. 在请求头中添加Authorization字段，值为token
  3. 发送请求
  4. 验证返回状态码为200

### 1.5 查询所有用户（测试接口）
- **请求方式**：GET
- **URL**：`http://localhost:8001/user/list`
- **参数**：无
- **预期响应**：

      {
        "userId": 1001,

  }

## 2. 视频服务接口

### 2.1 核心播放接口

### 2.1.1 获取视频播放详情
- **请求方式**：GET
- **URL**：`http://localhost:8002/play/detail/{videoId}`
- **参数**：
  - Path参数：`videoId`（视频ID）
- **预期响应**：
  ```json
  {
    "code": 200,
    "message": "操作成功",
    "data": {
      "videoId": 1,
      "title": "视频标题",
      "description": "视频描述",
      "coverUrl": "封面URL",
      "videoUrl": "视频URL",
      "duration": 35,
      "likeCount": 0,
      "coinCount": 0,
      "collectCount": 0,
      "shareCount": 0,
      "playUrl": "预签名播放地址",
      "realViewCount": 0
    }
  }
  ```
- **测试步骤**：
  1. 输入视频ID（如1、3、4、5）
  2. 发送请求
  3. 验证返回数据是否包含完整的视频信息和预签名播放地址

## 3. 弹幕服务接口

### 3.1 发送弹幕
- **请求方式**：POST
- **URL**：`http://localhost:8003/danmaku`
- **参数**：
  - Query参数：`userId`（用户ID，必填）
  - Body参数：
    ```json
    {
      "videoId": 1,
      "content": "这是一条测试弹幕",
      "videoTime": 10.5,
      "color": "#FFFFFF",
      "fontSize": 25,
      "mode": 1
    }
    ```
- **预期响应**：
  ```json
  {
    "code": 200,
    "message": "发送弹幕成功",
    "data": 1678901234567
  }
  ```
- **测试场景**：
  - 正常发送弹幕
  - 未传userId参数
  - 未传RequestBody参数
  - 视频ID不存在
  - 内容为空

### 3.2 获取视频的弹幕列表
- **请求方式**：GET

  ```json

    "data": [
      {
        "danmakuId": 1678901234567,
        "videoId": 1,
        "userId": 1,
        "content": "这是一条测试弹幕",
        "videoTime": 10.5,
        "color": "#FFFFFF",
        "fontSize": 25,
        "mode": 1,
        "nickname": "用户12345",
        "avatar": "https://example.com/avatar/default.jpg",
        "createTime": "2026-03-15T00:00:00"
      }
    ]
  }
  ```
- **测试场景**：
  - 获取指定视频的弹幕列表
  - 获取指定时间范围内的弹幕

## 5. 测试验证

### 5.1 Redis数据验证
使用Redis客户端执行以下命令检查数据：

```bash
# 检查点赞数
GET video:like:count:{videoId}

# 检查投币数
GET video:coin:count:{videoId}

# 检查收藏数
GET video:collect:count:{videoId}

# 检查分享数
GET video:share:count:{videoId}

# 检查用户点赞状态
EXISTS video:user:like:{userId}:{videoId}

# 检查用户收藏状态
EXISTS video:user:collect:{userId}:{videoId}

# 检查弹幕数量
GET danmaku:video:count:{videoId}

# 检查评论点赞状态
EXISTS comment:user:like:{userId}:{commentId}
```

### 5.2 MySQL数据验证
执行以下SQL查询检查数据：

```sql
-- 检查视频互动数据
SELECT video_id, like_count, coin_count, collect_count, share_count 
FROM t_video WHERE video_id = {videoId};

-- 检查用户数据
SELECT user_id, nickname, phone 
FROM t_user WHERE user_id = {userId};

-- 检查弹幕数据
SELECT * FROM t_danmaku WHERE video_id = {videoId};

-- 检查评论数据
SELECT * FROM t_comment WHERE video_id = {videoId};
```

### 5.3 接口数据一致性验证
1. 执行多个互动操作
2. 调用 `http://localhost:8002/play/detail/{videoId}` 获取视频详情
3. 验证返回的互动数据是否与Redis中的数据一致
4. 执行同步操作
5. 再次查询MySQL数据，验证与Redis数据一致

## 6. 测试顺序建议

1. 用户服务接口测试
   - 发送注册验证码
   - 用户注册
   - 用户登录
   - 查询用户信息
   - 用户退出登录

2. 视频服务核心播放接口测试
   - 获取视频详情
   - 上报播放量
   - 获取选集列表
   - 获取选集详情
   - 获取相关推荐

3. 视频服务互动接口测试
   - 点赞 → 取消点赞 → 再次点赞
   - 收藏 → 取消收藏 → 再次收藏
